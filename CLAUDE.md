# Claude Code 実装指示書:体重管理PWA(完成版まで一括実装)

このドキュメントをプロジェクトルートに `CLAUDE.md` として配置し、Claude Codeに実装を依頼する。
詳細な設計根拠は同梱の `weight-tracker-design.md` を参照(アルゴリズムの数式・データモデル・画面設計はすべてそちらが正)。

---

## 0. プロジェクト概要

体重管理PWA「MacroFactor代替アプリ」を作る。摂取カロリー実績と体重トレンドの差分からTDEE(消費カロリー)を逆算し、週次で目標摂取カロリーを自動調整する個人用ツール。

**このセッションのゴール**: 下記マイルストーン1〜3をすべて実装し、GitHub PagesにデプロイすればiPhoneで日常運用を開始できる完成状態にすること。

**技術方針**:
- 単一 `index.html`(HTML/CSS/JS全部入り、フレームワークなし、ES2020+)
- データ永続化は `localStorage` のみ、サーバー・外部API通信なし
- CDN・外部ライブラリ依存禁止(チャートはSVG自作)
- GitHub Pagesでホスティング
- iPhone Safari(ホーム画面追加してPWAとして使う)専用最適化

**既存プロジェクトとの関係**: 同ユーザーが運用中のバックギャモン学習PWA(GitHub Pages、単一HTML、iPhone Safari最適化済み)がある。viewport設定・タップ遅延対策・apple-touch-icon等のSafari最適化パターンを踏襲すること(コード共有はしない)。

- localStorageのキー名・スキーマは設計書3.1節に厳密に従う

## 3. マイルストーン2: アダプティブアルゴリズム + 週次見直し

### 3.1 TDEE推定(設計書4.2節)
```
estimatedTdee = avgIntake − (ΔtrendKg × kcalPerKgFat / periodDays)
```
- 直近 `tdeeWindowDays`(21日)のうち体重・摂取が両方記録された日を対象
- **ガード条件(必須)**: 記録10日未満→ `initialTdee` または前回値 / 週次変動±150kcalクランプ / 絶対範囲1600〜3200クランプ

### 3.2 目標算出(設計書4.3節・4.4節)
```
weeklyDeficitKcal = trendWeight × (weeklyLossRatePct/100) × kcalPerKgFat
targetAvgKcal     = estimatedTdee − weeklyDeficitKcal/7   // 下限1500クランプ
targetByDay[d]    = round(targetAvgKcal × dayMultipliers[d] / 5) × 5
// PFC: P = trendWeight × proteinPerKg(全曜日固定)、残カロリーをfatRatioで脂質、余りを炭水化物
```
- トレンド体重が `goalWeightKg` 以下になったら「維持モードへの切替(ペース0%)」を提案する表示を出す

### 3.3 週次見直しフロー(設計書4.5節)
- 週の起点は**月曜**(ローカルタイム)
- 新しい週に初めてアプリを開いた時、前週データで再計算→週次サマリモーダル表示(新TDEE、目標kcalの前週比、トレンド体重の変化)
- 計算結果は `wt_weekly` にキャッシュ(スキーマは設計書3.1節)
- ユーザーによる目標の手動上書きを可能にし、オーバーライドフラグを保持

### 3.4 「今週の残り」表示(今日タブに追加)
- 週の目標合計 −(今週の摂取実績合計)を残日数で割った1日あたり可処分カロリーを表示
- 1日超過を週内で吸収する運用に対応するための機能(ユーザーの管理スタイル上、重要度高)

### M2完了条件

## 4. マイルストーン3: 可視化 + PWA仕上げ

### 4.1 グラフタブ(SVG自作、ライブラリ不使用)
- **体重チャート**: 実測値(薄い点)+ EMAトレンド線(濃い線)+ 目標体重の水平線。期間切替: 1ヶ月/3ヶ月/全期間
- **TDEE推移**: 週次棒グラフ(`wt_weekly` から描画)
- **統計サマリ**: 直近4週の平均減量ペース(kg/週)、現ペースでの目標到達予測日
- タッチ操作は最小限でよい(タップで値表示があれば十分、ピンチズーム等は不要)

### 4.2 設定タブ
- 編集可能: 目標体重 / 週次ペース% / P per kg / 脂質比率 / 曜日別配分
- JSONエクスポート: 3キーをまとめてクリップボードコピー + ファイルダウンロード(`weight-tracker-backup-YYYYMMDD.json`)
- JSONインポート: テキストエリア貼り付け→検証→確認ダイアログ→復元
- 全データ削除(二段階確認)

### 4.3 PWA化
- インラインmanifest(またはmanifest.json): 名前・アイコン・`display: standalone`・テーマカラー
- `sw.js`: キャッシュファーストでオフライン動作
- apple-touch-icon、`viewport-fit=cover`、safe-area-inset対応
- ダークモード(`prefers-color-scheme`)

### 4.4 データ保全
- Safariの7日ITP対策として、最終エクスポートから30日経過したら「バックアップ推奨」バナーを表示

### M3完了条件
- [ ] グラフが実データで描画され、期間切替が動く
- [ ] ホーム画面追加→オフラインで起動できる
- [ ] エクスポート→全データ削除→インポートで完全復元できる
- [ ] ライト/ダーク両モードで視認性に問題がない

## 5. コーディング規約

- ES2020+、モジュール分割なし、単一ファイル内スコープ管理(IIFE可)
- 数値入力は `inputmode` 指定、`font-size` 16px以上(Safariズーム防止)
- 日付処理はすべてローカルタイム基準、`YYYY-MM-DD` 文字列で統一(Date型のタイムゾーン事故に注意)
- コメントは最小限、命名で意図を伝える

## 6. テスト観点(設計書7章より)
- EMA: 欠測日を挟んだ連続性
- TDEE: 極端値(絶食日/爆食日)でのクランプ動作
- 週境界: 月曜0時前後の入力が正しい週に属すること
- localStorage破損時にアプリがクラッシュせず初期化+警告表示すること
